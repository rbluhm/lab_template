# notes/

Tracked working memory for the project. Whenever an agent finishes a piece of
work, it **summarizes that work as a note here** — what was done, what was
found, what is still open.

- One note per topic/session: `YYYY-MM-DD_short-slug.md`.
- [`STATUS_OVERVIEW.md`](STATUS_OVERVIEW.md) is the index: it summarizes the
  current state and links every note, categorized into **Open** and
  **Resolved**. Always update it when you add or close a note.

This folder is for human-readable summaries and reasoning, not code or data.
Code gets promoted into `code/`; throwaway work stays in `sandbox/`.
